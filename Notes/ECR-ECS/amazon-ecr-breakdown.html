Amazon Elastic Container Registry (ECR)

Fully managed container image registry — complete service breakdown with all features, integrations, and capabilities.

What is Amazon ECR? Overview

Amazon ECR is an AWS managed container image registry service that is secure, scalable, and reliable. It supports private repositories with resource-based permissions using AWS IAM, so that specified users or Amazon EC2 instances can access your container repositories and images. You can use your preferred CLI to push, pull, and manage Docker images, OCI images, and OCI-compatible artifacts.

Fully managed

No infrastructure to provision or manage — AWS handles availability, scaling, and patching.

OCI compliant

Supports Open Container Initiative (OCI) standards and Docker Registry HTTP API V2.

AWS native

Deep integration with ECS, EKS, Lambda, Fargate, CodePipeline, and more.

S3-backed storage

All images are stored in Amazon S3 for high durability and availability.

Private Registry Core

Each AWS account gets a default private ECR registry per region. The URL follows the format https://aws_account_id.dkr.ecr.region.amazonaws.com.

Repositories

Named containers for Docker images, OCI images, and OCI-compatible artifacts like Helm charts and SBOMs.

Namespaces

Organize repositories by team, project, or workflow using slash-delimited namespace prefixes.

Image tag immutability

Prevent image tags from being overwritten or reassigned. Supports exclusion patterns for specific mutable tags.

Multi-arch images

Store multiple architecture variants (x86-64, ARM/Graviton) under a single image manifest list.

Image archiving

Archive images for long-term retention without needing to delete them outright.

Resource-based policies

Grant specific users, roles, or other AWS accounts access to individual repositories.

Public Registry (ECR Public) gallery.ecr.aws

Amazon ECR Public is a managed registry for publicly available container images. Images can be pulled anonymously or using an ECR Public authentication token — no AWS account required to browse or pull.

ECR Public Gallery

Public portal at gallery.ecr.aws listing OS images, AWS-published images, Kubernetes add-ons, and Helm charts.

Anonymous pulls

No authentication needed to pull public images — useful for open-source projects and public tooling.

AWS Marketplace containers

Verified, commercial container software available alongside open-source images in a single registry.

Free storage tier

50 GB of free storage per month, always. Transfers to AWS compute in any region are free and unlimited.

Public repo policies

Resource-based IAM policies can restrict who can push to public repositories while keeping pulls open.

Image catalog metadata

Add descriptions, usage instructions, and tags to your public repositories for discoverability.

Security IAM · KMS · Signing

ECR provides multiple layers of security covering access control, encryption at rest and in transit, and cryptographic image signing.

IAM access control

User and resource-based policies controlling push, pull, and admin actions per repository and per user/role.

Encryption at rest (SSE-S3)

Default encryption using Amazon S3-managed keys applied to all images automatically.

Encryption at rest (AWS KMS)

Bring-your-own customer master key (CMK) via AWS KMS for stronger compliance requirements.

Encryption in transit (HTTPS)

All image transfers are encrypted with TLS. Enforced by default — no insecure pull/push allowed.

Managed image signing

Set up signing rules via the console or a single API call. ECR automatically signs images on push using AWS Signer.

Notary client signing

Use the open-source Notary (cosign-compatible) client to manually create and push signatures with AWS Signer profiles.

Signing rules support custom signing profiles with configurable signature validity periods and per-principal IAM restrictions.
Image Scanning Vulnerability detection

ECR provides two scanning tiers — basic and enhanced — to detect vulnerabilities in container images before or after deployment.

Basic scanning

Uses the CVE database from the open-source Clair project. Scan on push or manually triggered.

Enhanced scanning

Powered by Amazon Inspector. Continuous, automated scanning for OS and programming language package vulnerabilities.

Scan on push

Configure any repository to automatically scan every image the moment it's pushed.

Image usage insights

Enhanced scanning surfaces where vulnerable images are actively running — on ECS clusters and EKS nodes — so you know actual exposure.

Scan findings are returned via the ECR API, console, and can be routed through Amazon EventBridge or Amazon Inspector's findings console.
Lifecycle Policies Storage management

Lifecycle policies let you define rules to automatically expire or archive images based on age or count, keeping storage costs under control without manual cleanup.

Rule-based expiry

Define rules by image count (keep latest N) or image age (expire after X days) per repository.

Archiving

Move images to long-term archive storage instead of deleting them outright. Retrieve when needed.

Policy preview

Preview which images a lifecycle policy will affect before applying it — no surprises.

CloudTrail auditing

Every lifecycle action (expire, archive) is recorded as a CloudTrail event for full audit history.

Actions are applied within 24 hours of images meeting expiration criteria. Policies can target specific image tags or untagged images.
Cross-Region & Cross-Account Replication Distribution

Registry-level replication automatically copies images to other AWS regions and/or other AWS accounts so images are wherever your workloads run.

Cross-region replication

Replicate images to one or more additional AWS regions. Configured as a registry-wide setting.

Cross-account replication

Share images across AWS accounts without manual pushes. Useful for hub-and-spoke multi-account setups.

Replication filters

Control which repositories and image tags are included in replication using filter rules.

Same repository name

Repository names remain identical across regions and accounts after replication — no renaming needed.

Only images pushed after replication is configured are replicated. Existing images and delete actions are not replicated — deletions must be performed in each registry manually.
Pull-Through Cache Upstream sync

Pull-through cache rules let you cache images from external upstream registries into your private ECR registry. ECR periodically syncs to keep cached images up to date.

Supported upstreams

Docker Hub, ECR Public, Kubernetes container registry, GitLab container registry, and any OCI-compliant registry.

ECR private as upstream

Another ECR private registry can also serve as the upstream source for pull-through cache rules.

Automatic sync

ECR periodically fetches updated images from upstream — no manual re-pulls needed to stay current.

Full ECR security stack

Cached repos get PrivateLink, KMS encryption, image scanning, cross-region replication, and lifecycle policies.

No throttling

Pull from the cached ECR copy instead of hitting upstream rate limits (e.g. Docker Hub pull limits).

Create-on-pull

Repositories are automatically created in your private registry the first time an image is pulled through.

Repository Creation Templates Automation

Templates let you define default settings for repositories that ECR creates automatically — during pull-through cache, replication, or create-on-push flows.

Tag immutability

Pre-configure whether auto-created repos enforce tag immutability.

Encryption config

Apply SSE-S3 or a specific KMS key to every auto-created repository.

Repository policies

Attach IAM resource policies to auto-created repos so access control is instant.

Lifecycle policies

Auto-attach lifecycle rules so new repositories immediately start managing their own storage.

Resource tags

Automatically tag new repositories for cost allocation, filtering, and compliance.

Networking — VPC Endpoints (PrivateLink) Private network

ECR supports AWS PrivateLink, allowing container image traffic to stay entirely within the Amazon network — no internet gateway, NAT device, or VPN needed.

Interface VPC endpoints

Access ECR APIs via private IP addresses inside your VPC using AWS PrivateLink.

IPv4 & IPv6 dual-stack

ECR endpoints support both IPv4-only and dual-stack (IPv4 + IPv6) configurations.

No internet required

All traffic between your VPC and ECR stays on the Amazon backbone — nothing traverses the public internet.

Endpoint policies

Attach IAM policies to VPC endpoints to restrict which actions and registries are accessible through that endpoint.

AWS Integrations ECS · EKS · Lambda · CI/CD

ECR integrates natively with the full AWS ecosystem — compute, orchestration, CI/CD, observability, and event-driven automation.

Amazon ECS

Specify ECR image URIs directly in ECS task definitions. ECR credentials are handled automatically via IAM task roles.

Amazon EKS

Use ECR images in Kubernetes pod specs. Node IAM roles authorize pulls without extra configuration.

AWS Lambda

Deploy Lambda functions packaged as container images stored in ECR — up to 10 GB image size.

AWS Fargate

Serverless container runs pull images from ECR directly. No node management, no credential setup needed.

AWS CodePipeline / CodeBuild

Build and push container images to ECR as part of fully managed CI/CD pipelines.

Amazon EventBridge

Route ECR events (image push, delete, scan findings) to Lambda, SNS, SQS, or Step Functions.

AWS CloudTrail

All ECR API calls are logged in CloudTrail — push, pull, delete, policy change — for full audit history.

Amazon Inspector

Provides the backend for enhanced scanning, delivering OS and language-level CVE findings continuously.

Third-party CI/CD

Compatible with GitHub Actions, GitLab CI, Jenkins, CircleCI, and any Docker-compatible toolchain.

Storage & Pricing Cost model

ECR uses a pay-as-you-go model. You pay for image storage and outbound data transfer. Most transfers within AWS are free.

Private free tier

500 MB of private storage per month free for the first 12 months under AWS Free Tier.

Public free tier

50 GB of public storage per month, always free — no 12-month limit.

Free intra-AWS transfers

Data transfer between ECR and ECS, EKS, Fargate, Lambda, or App Runner in the same region costs nothing.

Pay-per-use storage

Charged per GB-month of private image data stored beyond the free tier. No upfront costs.

Enhanced scanning costs are based on the number of images scanned, billed through Amazon Inspector. Standard ECR scanning (basic) is free.
