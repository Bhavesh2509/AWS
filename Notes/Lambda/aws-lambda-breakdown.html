🟠 AWS Lambda – Complete Breakdown

What is Lambda? AWS Lambda is a serverless, event-driven compute service. You write code (a "function"), Lambda runs it in response to triggers. No servers to provision or manage. You pay only for actual execution time, billed per millisecond.

1. Fundamental Capabilities

Automatic Scaling

Lambda scales to match incoming request rates and scales to zero during idle periods — no manual configuration needed. Each function can scale up to 1,000 concurrent executions every 10 seconds, up to the account concurrency limit.

Native Service Integration

Integrates natively with 220+ AWS services and 50+ SaaS apps — no custom integration code needed.

Built-in Fault Tolerance

Lambda runs across multiple Availability Zones. Durable Functions adds built-in error handling, automatic retries, stateful multi-step workflows (up to 1 year), and checkpointing — so failed steps resume from where they left off.

3. Security & Governance

Built-in Secure Isolation

Each function runs inside a Firecracker MicroVM on the AWS Nitro System — hardware-level isolation, never shared across functions, customers, or accounts.

  • Tenant isolation mode: Each end-user invocation runs in a separate environment — critical for SaaS
  • VPC integration: Place Lambda inside your private VPC to access RDS, ElastiCache, internal APIs
  • IAM execution roles: Fine-grained control over what AWS services each function can call
  • Secrets Manager: Never hardcode credentials

4. Performance Optimization

Provisioned Concurrency

Pre-warms N function instances so they're always ready — eliminates cold start latency. No code changes required. You pay for pre-warmed instances even when idle (cost vs. latency trade-off). Can be scaled up/down on demand.

SnapStart

Snapshots the initialized execution environment after the init phase. On cold starts, restores from snapshot instead of initializing from scratch.

  • Java: cold starts reduced up to 10x (e.g., ~5s → sub-second)
  • Python and .NET: from several seconds to sub-second
  • No code changes required for most functions

Provisioned Mode for Event Source Mappings (ESM)

Pins dedicated resources to Kafka event source connectors (Amazon MSK or self-managed Kafka), solving throughput and latency bottlenecks in high-volume streaming pipelines.

5. Cost Control

x86 vs. ARM (Graviton2) Architecture

Switch between x86 and ARM with a single config change — no code rewrite. ARM is ~20% cheaper per GB-second and often faster for most stateless workloads.

Lambda Managed Instances (EC2 Capacity)

Run Lambda functions on specific EC2 instance types (e.g., GPU for ML, high-memory for data). AWS still manages all patching, routing, load balancing, and autoscaling. Keeps the familiar Lambda programming model while giving EC2-level pricing predictability.

6. Pricing

Free Tier (permanent, not just first 12 months):

  • 1 million requests/month free
  • 400,000 GB-seconds of compute time/month free

Beyond free tier: ~$0.20–$0.28 per additional million requests (varies by region). Compute billed per GB-second (memory × duration).

⚠️ If Lambda runs inside a VPC and needs internet access, a NAT Gateway is required — adds ~$0.045/hr in US East. This is often the largest hidden cost.
