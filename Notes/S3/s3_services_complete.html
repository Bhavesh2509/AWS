S3 Block Public Access

Four settings applied at account or bucket level: BlockPublicAcls, IgnorePublicAcls, BlockPublicPolicy, RestrictPublicBuckets. When enabled they override every other permission — bucket policies, ACLs, nothing can make the bucket public. Enabled by default on all new buckets since April 2023. AWS Trusted Advisor and IAM Access Analyzer surface any bucket deviating from this.

close ✕

IAM policies & S3 bucket policies

Two complementary layers — both must allow an action for it to succeed. IAM policies are identity-based (attached to users, roles, groups). Bucket policies are resource-based (attached to the bucket, enable cross-account access). IAM Access Analyzer for S3 continuously evaluates policies and surfaces findings for public or unintended cross-account access with one-click remediation.

close ✕

Access Control Lists (ACLs)

Legacy per-object access mechanism. AWS now recommends disabling ACLs entirely using S3 Object Ownership (Bucket owner enforced setting). Once disabled, all objects are owned by the bucket owner and access is managed exclusively via bucket policies and IAM — simpler and easier to audit.

close ✕

Enforced org-wide via Service Control Policies.

Multi-Region Access Points (MRAP)

A single global S3 endpoint that routes requests to the lowest-latency bucket across multiple AWS regions over the AWS global network backbone (not the public internet). Supports active-active configurations and automatic failover with Cross-Region Replication. Reduces global access latency by ~60% vs region-specific endpoints.

Encryption

Three server-side modes: SSE-S3 (AWS-managed AES-256, default since 2023, no extra cost), SSE-KMS (customer-managed keys in AWS KMS with full CloudTrail audit trail and key rotation), SSE-C (customer provides the key per request — AWS encrypts and immediately discards the key). Client-side encryption also supported. TLS enforced in transit.

S3 Object Lock (WORM)

Write-once-read-many protection at object or bucket level. Compliance mode: no one — not even root — can delete before the retention period expires. Meets SEC 17a-4, FINRA, CFTC requirements. Governance mode: authorized users with a special IAM permission can override. Legal Hold: indefinite lock ignoring any retention date. Works with versioning to protect each version independently.

close ✕

Pre-signed URLs

Generate a URL embedding temporary AWS credentials for a specific S3 operation, valid for seconds up to 7 days. Anyone with the URL can perform that one operation (GET, PUT, DELETE) without needing their own AWS credentials. Common uses: sharing private files with customers, direct browser-to-S3 uploads without proxying through your server, temporary media streaming access.

close ✕

VPC endpoints & Amazon PrivateLink for S3

Gateway VPC endpoints route S3 traffic over the AWS backbone — free, requires only a route table entry. Interface VPC endpoints (PrivateLink) create private IPs inside your VPC, enabling on-premises systems to reach S3 over Direct Connect without touching the internet. Security groups and endpoint policies restrict which buckets are reachable through each endpoint.

S3 Versioning

Once enabled, every PUT creates a new version with a unique ID. DELETE operations insert a delete marker — data is not immediately destroyed. MFA Delete can require a second authentication factor before permanent deletion. Lifecycle rules can expire non-current versions to control storage costs. Prerequisite for Cross-Region Replication and S3 Object Lock.

close ✕

S3 Replication

Three modes: CRR (Cross-Region Replication) for geo-redundancy, compliance, and lower latency for global users; SRR (Same-Region Replication) for test/prod separation or log aggregation; Batch Replication for backfilling existing objects to a new destination. Copies metadata, tags, and ACLs. S3 Replication Time Control (RTC) guarantees 99.99% of objects replicated within 15 minutes with CloudWatch metrics.

S3 Batch Operations

Manage billions of S3 objects at scale with one API call. Supported operations: Copy, Invoke Lambda, Restore from Glacier, Tag objects, Replace ACLs, Apply Object Lock, Replicate. Accepts an S3 Inventory report or custom CSV as a manifest. Handles retries automatically, tracks progress with real-time metrics, and produces detailed completion reports. At re:Invent 2025, AWS announced 10x performance improvements for large batch jobs.

close ✕

Lifecycle policies

Rules that automate transitions and expirations: move objects to cheaper storage after N days, expire current versions, permanently delete non-current versions, or abort incomplete multipart uploads. Filter by prefix, object tags, or size range. Example chain: Standard → Standard-IA (30 days) → Glacier Instant (90 days) → Deep Archive (365 days) → Delete (2555 days).

close ✕

Multipart upload & 50 TB max object size

Objects over 100 MB should use multipart upload; objects over 5 GB must. Parts upload in parallel for maximum throughput. Failed parts retry independently without restarting the whole upload. At re:Invent 2025, AWS raised the maximum S3 object size from 5 TB to 50 TB — a 10x increase enabling 8K RAW video, complete scientific simulation outputs, and large AI training corpora as single atomic objects.

S3 Transfer Acceleration

Routes uploads to the nearest CloudFront edge location then across the optimized AWS backbone to the target S3 bucket. Speed improvement is greatest for objects 1 GB+, transfers from geographically distant clients, or high-latency networks. Enabled per-bucket with a different accelerate endpoint URL. Use the S3 Speed Comparison tool to verify improvement before enabling.

Metrics include: storage by class, bucket, prefix, account; data transfer, replication lag, non-current version accumulation, incomplete multipart uploads, encryption coverage. S3 Inventory

Scheduled flat-file reports (daily or weekly) listing all objects or a filtered subset in CSV, ORC, or Parquet. Fields include: key, size, last modified, ETag, storage class, encryption status, replication status, version ID, delete marker. Used as input manifests for S3 Batch Operations. Essential for compliance audits verifying encryption coverage and data ownership.

S3 Event Notifications

Configure notifications for: ObjectCreated (Put, Post, Copy, CompleteMultipartUpload), ObjectRemoved, ObjectRestore, Replication events, Lifecycle transitions, and Intelligent-Tiering archival. Destinations: SNS, SQS, Lambda functions, and Amazon EventBridge. EventBridge integration enables filtering and fan-out to Step Functions, CodePipeline, and hundreds of downstream services without custom glue code.

close ✕

CloudTrail data events for S3

Records every S3 API call — GetObject, PutObject, DeleteObject, ListBucket, etc. — with caller identity, source IP, timestamp, and full request parameters. Critical for SOC 2, PCI DSS, and HIPAA compliance and forensic investigation after a security incident. Filter by bucket prefix to control cost. Logs delivered to S3 bucket or CloudWatch Logs.

close ✕

CloudWatch metrics for S3

Daily storage metrics free for all buckets. Optional per-request metrics (paid, 1-minute granularity): 4xx/5xx error rates, first-byte latency, total request latency, upload and download throughput. Filter by prefix or tag for cost allocation. Set alarms on estimated charges to prevent billing surprises.

close ✕

Server access logging

Generates detailed log records for every request: bucket name, requester, operation, object key, HTTP status, bytes sent, error code, response time, user agent. Delivered best-effort to a target S3 bucket. Used for: security auditing, understanding customer access patterns, analyzing S3 cost by operation type, detecting unauthorized access.

Snowball Edge: 80–210 TB with on-device compute for processing before shipping. Snowmobile: a 45-foot shipping container on a truck holding up to 100 PB — used for complete data center migrations. Devices ship to AWS and data loads directly to S3.

Direct Connect + S3 PrivateLink

Direct Connect provides a dedicated 1–100 Gbps private fiber connection from your data center to an AWS Direct Connect location. Combined with PrivateLink interface VPC endpoints for S3, you get a fully private path from on-premises to S3 — traffic never touches the public internet. Consistent bandwidth and latency. Required for compliance scenarios mandating full network isolation.

close ✕

AWS DataSync

Fully managed service for automating data movement between on-premises file systems (NFS, SMB, HDFS) and S3. Handles scheduling, monitoring, retries, and data integrity verification automatically — up to 10x faster than rsync or custom scripts. Also transfers between S3 and EFS, S3 and FSx, and between two S3 buckets.

close ✕

Mountpoint for S3

Open-source FUSE file client that mounts an S3 bucket as a local POSIX-like file system on Linux. Optimized for read-heavy sequential access (ML training, media processing). Limitations: no random writes, no rename support — not a general POSIX filesystem. Supports S3 Express One Zone for maximum throughput on ML workloads.
